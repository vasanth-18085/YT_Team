## COIN Framework — Audience Language

### Audience Language (How they talk)
| Term They Use | What They Mean |
|--------------|----------------|
| "algo trading" | Algorithmic trading broadly |
| "quant" | Quantitative finance / quant finance |
| "ML for stocks" | Machine learning applied to financial markets |
| "backtest" | Historical simulation of a strategy |
| "alpha" | Return above the market, edge |
| "NIFTY" | NSE NIFTY 50/200 index |
| "yfinance" | The Python library for free stock data |

| "from scratch" | Without relying on libraries (shows real understanding) |
| "real data" | As opposed to toy/example data — NSE actual data |
| "walk-forward" | Out-of-sample validation over rolling time windows |
| "overfitting" | Model fits training data, fails on new data |
| "signal" | A systematic predictor of returns |

### Tone Rules
1. **No hype-face.** Thumbnails show honest expressions. Not "shocked face at screen."
2. **No guaranteed returns language.** Ever. Not even "this works."
3. **Failures are content.** Walk-forward that cut the Sharpe in half is a video, not a confession.
4. **Admit uncertainty.** "I'm not sure this is the best approach" is fine and builds trust.
5. **NSE/NIFTY primary.** Every project built on Indian market data first. US is secondary.
6. **Code-first, theory second.** Show the implementation, then explain what it's doing.
