## SECTION 2 — THE PROBLEM: PREDICTIONS WITHOUT CONFIDENCE (2:00–6:00)

Here is the issue with every standard forecasting setup.

Your model says: "I predict stock will go up." It does not say: "I am 75% sure" or "I am barely better than a coin flip." The prediction comes out looking the same regardless of whether the model is in a regime it understands well or completely outside its training distribution.

Meta-labeling solves this with a very clean framing: build a second model that learns to predict — not the direction of the stock — but whether the first model's prediction will be right.

[INFORMATION GAIN] The idea comes from Marcos Lopez de Prado's work in "Advances in Financial Machine Learning." The core insight is that primary models have uneven accuracy across different market conditions. In trending markets, momentum-based models work extremely well. In choppy, mean-reverting markets, they fail. The meta-classifier can learn to recognise which regime it is in from features like volatility, sentiment alignment, and recent prediction track record, and use that to decide whether to trust the primary model right now.

Concrete example:
- Primary model: "AAPL will go up" → Meta-classifier: 85% confident this prediction is correct → Trade it
- Primary model: "TSLA will go up" → Meta-classifier: 52% confident → Skip this trade, too close to a coin flip

## SECTION 4 — STAGE 2: GENERATING META-LABELS (8:00–14:00)

The meta-labels are generated by comparing the primary model's prediction direction to the ground truth outcome from the triple-barrier labels in Video 3.

```python
def create_metalabels(forecasts, ground_truth_labels):
    # forecasts: array of signed predictions from TiDE
    # ground_truth_labels: +1 (take-profit), -1 (stop-loss), 0 (timeout) from triple-barrier

    metalabels = []
    for i, forecast in enumerate(forecasts):
        ground = ground_truth_labels[i]

        if ground == 0:              # Timeout: outcome ambiguous, exclude from training
            metalabels.append(np.nan)
        elif (forecast > 0 and ground == 1) or (forecast < 0 and ground == -1):
            metalabels.append(1)     # Primary model was directionally correct
        else:
            metalabels.append(0)     # Primary model was wrong

    return metalabels
```

[INFORMATION GAIN] Timeout cases — where neither take-profit nor stop-loss was hit — get assigned NaN, not 0. This is important. If you label timeouts as "primary model was wrong" (0), you are training the meta-classifier to penalise predictions on days where nothing happened. That is the wrong supervision signal. The meta-classifier should only learn from definitive outcomes. Timeouts are excluded from training.

In practice, with the barrier parameters I use — 2% take-profit, 1% stop-loss, 20-day max hold — approximately 30% of observations are timeouts. The training set after exclusion is still large enough for meaningful learning.

## SECTION 5 — STAGE 3: META-CLASSIFIER FEATURES (14:00–20:00)

The meta-classifier does not see the raw price series. It sees features specifically chosen to predict whether the primary model's current prediction is reliable.

```python
def build_meta_features(X_test, forecasts, returns, volume):
    meta_features = []
    for i in range(len(X_test)):
        # Prediction strength from primary model
        raw_pred = float(forecasts[i])

        # Has the primary model been accurate recently?
        recent_accuracy = compute_rolling_accuracy(forecasts, labels, i, window=20)

        # Is the market calm or turbulent?
        return_vol = np.std(returns[max(0, i-21):i])

        # Does sentiment corroborate the forecast direction?
        sentiment_alignment = check_sentiment_alignment(forecasts[i], sentiment_df, i)

        # Is volume confirming the move?
        vol_ratio = volume[i] / np.mean(volume[max(0, i-20):i])

        meta_features.append([
            raw_pred,
            recent_accuracy,
            return_vol,
            sentiment_alignment,
            vol_ratio
        ])
    return np.array(meta_features)
```

[INFORMATION GAIN] Why these five features specifically? Each one addresses a known failure mode of the primary model. `recent_accuracy` captures regime persistence — when a model is on a hot streak, it is in a market regime it was designed for; when it starts failing, the regime has likely shifted. `return_vol` captures market turbulence; the primary model is a trend follower, and it fails in choppy, high-volatility environments. `sentiment_alignment` captures multi-signal agreement; when FinBERT and the price model point the same direction, that agreement increases the prior probability of a correct prediction. `vol_ratio` captures volume confirmation — breakouts on thin volume often reverse.

## SECTION 6 — STAGE 4: NINE CLASSIFIERS (20:00–28:00)

The meta-classifier is not a single model. Nine architectures are trained and compared.

```python
class MetaClassifier:
    def __init__(self):
        self.classifiers = {
            'LightGBM':        LGBMClassifier(n_estimators=100),
            'XGBoost':         XGBClassifier(n_estimators=100),
            'CatBoost':        CatBoostClassifier(iterations=100, verbose=0),
            'RandomForest':    RandomForestClassifier(n_estimators=100),
            'LSTM':            LSTMClassifier(hidden_size=32, seq_len=10),
            'CNN':             CNNClassifier(in_channels=5, kernel_size=3),
            'TabNet':          TabNetClassifier(n_d=16, n_a=16, n_steps=5),
            'FT-Transformer':  FTTransformerClassifier(d_token=32, n_blocks=3),
            'Stacking':        StackingClassifier(
                                   estimators=[('lgb', LGBMClassifier()),
                                               ('xgb', XGBClassifier()),
                                               ('tabnet', TabNetClassifier())],
                                   final_estimator=LogisticRegression()
                               )
        }

    def fit(self, meta_features, metalabels):
        for name, clf in self.classifiers.items():
            clf.fit(meta_features, metalabels)

    def predict_proba_all(self, meta_features):
        return {name: clf.predict_proba(meta_features)[:, 1]
                for name, clf in self.classifiers.items()}
```


[INFORMATION GAIN] Two things to notice in this table. First, Stacking wins on AUC but does not win on Recall. FT-Transformer has the second highest AUC but is over-confident — its probability scores are systematically too high, which matters for position sizing. A classifier that says "90% confident" when it is actually right 75% of the time will cause you to over-size positions. Second, RandomForest has the highest Recall — it finds the most true positives — but also has the lowest Precision. In a capital-limited system, Precision matters more. You would rather miss some good trades than take many bad ones.

## SECTION 7 — FILTERING TRADES WITH CONFIDENCE THRESHOLDS (28:00–33:00)

Once the meta-classifier is trained, it produces a probability score for each primary prediction. That score becomes a confidence gate.

```python
def generate_trade_signals(primary_forecasts, meta_confidence, threshold=0.70):
    trades = []
    for pred, conf in zip(primary_forecasts, meta_confidence):
        if conf > threshold:
            action = 'BUY' if pred > 0 else 'SELL'
            size = position_size(conf)   # scale size proportionally to confidence
            trades.append({'action': action, 'size': size, 'confidence': conf})
        else:
            trades.append({'action': 'HOLD', 'size': 0, 'confidence': conf})
    return trades
```

### The performance impact

```
Without meta-labeling: 1,000 trades/year,  win rate 53%, Sharpe 0.71
With meta-labeling (conf > 0.70): 400 trades/year, win rate 68%, Sharpe 1.15
```

[INFORMATION GAIN] You lose 600 trades per year — 60% of your trade volume. You gain 15 percentage points of win rate and a 0.44 improvement in Sharpe. The intuition is clear: the 600 trades you are skipping are the model's low-conviction guesses. Removing them improves the signal-to-noise ratio of what remains. Transaction costs also fall proportionally — 400 trades per year means lower cumulative spread and commission drag.

The sizing rule `position_size(conf)` is also meaningful. A 75% confidence prediction gets a full position. An 85% confidence prediction gets 1.5x. This is not the Kelly criterion (covered in Video 17) but a linear scaling that increases capital exposure proportionally to the model's own confidence estimate.

## SECTION 8 — AVOIDING OVERFITTING IN META-CLASSIFIERS (33:00–38:00)

Meta-classifiers are especially prone to overfitting because they are trained on prediction errors — which have their own temporal autocorrelation. If the primary model makes mistakes in clusters (which it does, in difficult market regimes), and your meta-classifier's training set includes those cluster periods, it will overfit to the specific autocorrelation pattern of that regime.

The solution: nested cross-validation.

```python
for outer_fold in range(n_outer):
    X_train_outer, X_val_outer, X_test_outer = get_outer_fold_splits(outer_fold)

    # Inner loop: train primary model, generate predictions on validation
    primary.fit(X_train_outer)
    primary_preds_val = primary.predict(X_val_outer)
    meta_labels_val = create_metalabels(primary_preds_val, y_val_outer)

    # Train meta-classifier ONLY on validation-fold predictions
    meta.fit(build_meta_features(X_val_outer, primary_preds_val), meta_labels_val)

    # Evaluate on completely held-out test fold
    primary_preds_test = primary.predict(X_test_outer)
    meta_probs_test = meta.predict_proba(
        build_meta_features(X_test_outer, primary_preds_test)
    )
    evaluate(primary_preds_test, meta_probs_test, y_test_outer)
```

[INFORMATION GAIN] The critical constraint: the meta-classifier trains on the *validation* set predictions, not the training set. If you train the primary model on data, then train the meta-classifier on the primary's predictions from the same data, the meta-classifier is learning which patterns in the training set caused errors — and those patterns will not repeat identically. Nested CV ensures the meta-classifier trains only on out-of-sample primary predictions.

**Class imbalance:** If the primary model is 53% accurate, the meta-labels are roughly 53% "correct" and 47% "wrong" — mild imbalance. Use SMOTE (Synthetic Minority Oversampling Technique) or focal loss to prevent the classifier from defaulting to always predicting "correct."
